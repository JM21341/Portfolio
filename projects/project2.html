Food Ordering System

A collaborative final project for Intermediate Programming

About the Project

This project was created by five people with John Miguel L. Fetalcorin being the one who leads his members. They used Git and GitHub as a way to collaborate in doing the project, with some rules made for pushing and merging. They must first ask for the admin if there are any pulls required before they create another branch. Until then, the admin will tell his cooperative to not do anything at the moment or use a different temporary file to test. Once the other is finished, the admin requires everyone to pull after he merges the pull request that member made.

Our Food Ordering System is a simple C++ program designed to make ordering food easier and more organized. It allows users to view the menu, choose their meals or combo deals, place orders, and process payments.

The system also saves order records and generates reference numbers for easier tracking. This project was created to provide a convenient ordering experience while showcasing the use of C++ programming, file handling, and basic system management.

Functional Requirements
  1. Main Menu - This shows the main menu and the available options in the program.
  2. Add Order - An operation for adding an order.
  3. Remove Order - The user can remove their order via this function.
  4. View Order - The user can view their order by selecting this module in the main menu.
  5. Checkout Order - After ordering, the user can checkout. The user will then be asked to pay cash or cashless.
  6. Saving Changes to Files - The program will automatically save their order in a .csv file type record. The changes in the stock of foods will also be saved by automatically updating the file.
File Requirements
  1. structs.h - contains all the data structure that will be used in the program.
  2. file_handler.h - contains the function prototypes for the file operations in the program.
  3. order_handler.h - contains all the prototypes for handling orders.
  4. payment.h - includes all the function declarations for the payment process
  5. util.h - includes utility functions.
  6. main.cpp - the main program.
  7. file_handler.cpp - defines the functions from the file_handler header file.
  8. order_handler.cpp - contains the implementation of handling orders.
  9. payment.cpp - defines all the functions in the payment header file.
  10. util.cpp - contains the declaration for the utility functions used in this program.
  11. foods.csv - contains all the data for the food, including its ID, the combo meal details, price, and its stock.
  12. orders.csv - includes all the orders pending as of the moment.
